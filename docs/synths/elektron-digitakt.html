Fill Mode WTF

  1. Press YES and PAGE to activate fills for the next repetition of the pattern

  2. Hold PAGE to activate fills immediately while held down

  3. To latch fill mode, you must be stopped!

  4. Then, press YES and PAGE, release PAGE, then release YES

  5. Press PAGE to unlatch Fill Mode (you can do this while playing)

LFO Page WTF

  1. Set the Speed to 8

  2. Set the MULT to 16

  3. This will produce one cycle for each page, assuming the use of 1⁄16 notes per step

  4. I find the relationship confusing

    SPD⁄MULT 2 8 32 128
    8 128 32 8 2
    32 32 8 2 1⁄2
    64 16 4 1 1⁄4

  5. For ‚ÄúFAD‚Äù a negative value is a fade-in

  6. The ‚ÄúMODE‚Äù values are as follows:

    Value Meaning
    FRE Free-running, never restarting on triggers (though seems to restart on play)
    TRG Restarts when a note is triggered
    HLD Free-running, but a trigger latches to the current value until the next trig
    ONE Runs once, like an envelope
    HLF Runs for half the wave, then stops, like an envelope

Ratchet or Flam Steps (Retrig)

  1. Press and hold any TRIG keys

  2. Press ↑ or ↓ to open menu

  3. ‚ÄúRetrig‚Äù must be on

  4. Press ↑ or ↓ to change the rate of retriggering

  5. Turn D to configure how long retriggering will happen

  6. Turn H to adjust the velocity curve. Negative values will fade out, positive fade up.

MIDI trigs cannot be ratcheted. Also note if a trig occurs before the rate + length would complete, the retriggering will be stopped by the trig

Time Division

  1. Press FUNC and PAGE to enter the scale menu

  2. Change scale as needed. 1x are 1⁄16 notes, 2x are 1⁄32 notes, 1⁄2x are 1⁄8 notes, etc.

Trig Conditions WTF

  1. Hold down TRIG step(s) to lock trigs, or don't to affect track

  2. Choose the condition

    Condition Meaning
    X% Play X% of the time
    FILL Played only when in fill mode
    FILL Played when not in fill mode
    A:B Trigger A times out of B repeats of the pattern
    PRE Play if most previous condition was true
    PRE Played when most previous condition not true
    1ST Play only the first time the pattern is looped
    1ST Play always, exception the first time the pattern is looped
    NEI Play if most previous condition on track-1 was true
    NEI Played when most previous condition on track-1 not true

To trig different notes on the same step under different conditions, micro-timing can be used to move any step completely forward or backward one step

Notes and Non-Obvious Behaviors

  • A Sound is a sample + the configuration of Digitakt parameters, like a preset

  • Almost everything (including tempo and audio routing) is per pattern, so add patterns by copying one with the settings you want

  • Program change trigs won't work if MIDI → SYNC → PR CH SEND is checked

  • Also note that program changes are handled by external hardware differently, so they may simply queue a program change for the next bar

  • MIDI Outputs for the audio channels send CC only, not notes
